compiler-no-in-package-warning

In previous releases, this condition was signaled when a file without an in-package form was compiled. (Such fasl files could not be libfasl loaded using the Allegro Presto facility.) In 7.0, Allegro Presto is no longer supported. This condition is no longer signaled under any circumstances, but the class is still defined for backward compatibility (since handlers will error if they are defined with non-existent condition classes). See The Allegro Presto facility has been removed in loading.htm for further information.
