rsa-encrypt

Arguments: value key

RSA encrypt value using key. key can be a public or private key. (Recall that keys are vectors with three values: t/nil (distinguishing public from private but not used in actual encryption); the modulus value; and the exponent value. See Support for RSA encryption in miscellaneous.htm for details. Possible keys are returned by the function generate-rsa-keys.)

value can be:

  • an array of element-type (unsigned-byte 8).
  • a string (which is converted to an array of (unsigned-byte 8) using the :utf-8 external format).
  • an integer less than the modulus of the key.

The result of encryption will be an (unsigned-byte 8) vector (also called an octet vector) if value is a string or octet array and it will be an integer if value is an integer.

See rsa-decrypt and generate-rsa-keys.

See Support for RSA encryption in miscellaneous.htm for information on RSA encryption and RSA encryption support in Allegro CL. There are examples of RSA encryption and decryption in that section. Support for encryption in the same document discusses encryption support in Allegro CL in general.
