*jlinker-set-accessible*

The value of this special variable determines the jLinker behavior when Java throws java.lang.IllegalAccessException during a method call.

  • If the value of the variable is non-nil (the default), then jLinker calls m.setAccessible(True) where m is the method in the failed call, and then attempts the failed call one more time.
  • If the value of the variable is nil, then jLinker signals an error. One of the restarts available at this point is to retry the call after calling m.setAccessible(True).

See Calling Methods of Inner Classes in jlinker.htm for a more detailed discussion of this behavior.

See jlinker.htm for more information on the jLinker facility.
